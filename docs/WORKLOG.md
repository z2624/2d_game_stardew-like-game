## 2026-03-06

### 决策
1. **技术栈选定 Phaser 3**
   - 理由: 纯代码驱动，无编辑器负担，2D 俯视角支持完善
   - 替代方案: 原生 Canvas（太底层）、Godot（需要编辑器操作）

2. **3D→2D 转换方案确认**
   - 视角: 2D 俯视角，角色自动面向移动方向
   - 地形: 假透视单层地图（星露谷式）
   - 日落检测: 角色朝向 + 观景点双重判定
   - 美术: 代码生成占位图起步，后期替换

3. **协作模式确定**
   - 飞书: 日常沟通、需求确认
   - VSCode: 本地查看代码、目录规划
   - GitHub: 代码同步、版本控制

### 问题与解决

#### 问题 1: 空文件夹无法被 Git 跟踪
- **描述**: 创建的 `src/`、`scenes/` 等空文件夹没有同步到 GitHub
- **解决**: 在每个空文件夹中添加 `.gitkeep` 占位文件
- **状态**: ✅ 已解决

#### 问题 2: GitHub 推送需要认证
- **描述**: 服务器 push 到 GitHub 需要用户名密码
- **解决**: 使用 Personal Access Token 配置远程仓库 URL
- **状态**: ✅ 已解决
- **参考**: `git remote set-url origin https://username:token@github.com/...`

#### 问题 3: clawhub 限流导致 skills 无法安装
- **描述**: `clawhub install` 频繁返回 "Rate limit exceeded"
- **解决**: 暂时搁置 skills，直接用 Phaser 3 核心库开发
- **状态**: 🔄 进行中（等待限流恢复后可补装）
- **影响**: 无阻塞，Phaser 3 无需特殊 skills

#### 问题 4: 3D→2D 转换的关键设计决策
- **描述**: 原 PRD 基于 Three.js 3D，需要全面调整为 2D 俯视角
- **解决**: 创建 `PRD_2D_Adjustment.md` 文档，逐项标记变更
- **状态**: ✅ 已解决（文档待审阅）
